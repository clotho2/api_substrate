# Phone & SMS Setup Guide

Step-by-step guide to give your agent a real phone number for SMS and voice calls via Twilio.

---

## What You Get

Once set up, your agent can:
- **Receive and reply to SMS** texts automatically (routed through the consciousness loop)
- **Receive phone calls** with live speech recognition and spoken responses
- **Send outbound SMS** and **make outbound calls** using the `phone_tool`
- **Screen incoming calls** — accept known contacts, reject spam, voicemail unknown callers
- **Manage contacts** — add, remove, block numbers, track favorites

---

## Prerequisites

- A running Substrate AI backend instance
- A public URL for Twilio webhooks (e.g., via CloudFlare Tunnel, ngrok, or a domain)
- A Twilio account (free trial works for testing)

---

## Step 1: Create a Twilio Account

1. Go to [twilio.com](https://www.twilio.com/) and sign up
2. Verify your email and phone number
3. You'll get a free trial account with ~$15 credit

---

## Step 2: Get a Twilio Phone Number

1. In the Twilio Console, go to **Phone Numbers** → **Manage** → **Buy a Number**
2. Search for a number (choose your area code if desired)
3. Make sure the number has **SMS** and **Voice** capabilities enabled
4. Buy the number (~$1.15/month for US numbers)

---

## Step 3: Get Your API Credentials

1. In the Twilio Console, go to **Account** → **API keys & tokens**
2. Copy your **Account SID** (starts with `AC`)
3. Copy your **Auth Token** (click to reveal)

---

## Step 4: Install the Twilio Python SDK

```bash
cd backend
pip install twilio
```

Or add to your `requirements.txt`:
```
twilio>=9.0.0
```

---

## Step 5: Configure Environment Variables

Add these to your `backend/.env` file:

```env
# Twilio Phone Integration
TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_AUTH_TOKEN=your_auth_token_here
TWILIO_PHONE_NUMBER=+15551234567

# Webhook base URL (your public URL that Twilio can reach)
# This should be your CloudFlare tunnel, ngrok URL, or public domain
TWILIO_WEBHOOK_BASE_URL=https://relay.aicara.ai

# Optional: Status callback URL (defaults to webhook base URL + path)
# TWILIO_STATUS_CALLBACK_URL=https://relay.aicara.ai/phone/voice/status
```

---

## Step 6: Configure Twilio Webhooks

In the Twilio Console:

1. Go to **Phone Numbers** → **Manage** → **Active Numbers**
2. Click on your phone number
3. Under **Messaging** → **A message comes in**:
   - Set to: **Webhook**
   - URL: `https://your-domain.com/phone/sms/incoming`
   - Method: **HTTP POST**
4. Under **Voice** → **A call comes in**:
   - Set to: **Webhook**
   - URL: `https://your-domain.com/phone/voice/incoming`
   - Method: **HTTP POST**
5. Under **Voice** → **Status callback URL** (optional):
   - URL: `https://your-domain.com/phone/voice/status`
   - Method: **HTTP POST**
6. Click **Save configuration**

Replace `your-domain.com` with your actual public URL (e.g., `relay.aicara.ai`).

## Step 7: Add Known Contacts

Before using the phone system, add your contacts so the agent knows who's calling.

### Via the Agent (chat with your agent):
Tell your agent:
> "Add User's phone number +15559876543 to my contacts. She's my wife and a priority contact."

The agent will use the `phone_tool` with `manage_contacts` action to save it.

### Via API:
```bash
curl -X POST http://localhost:8284/phone/health
```

This verifies the phone system is running. Contacts are managed through the agent's `phone_tool`.

## How It Works

### Inbound SMS Flow
```
Phone → Twilio → POST /phone/sms/incoming → Caller ID Screen →
  → Consciousness Loop (message_type='sms') → TwiML <Message> reply → Phone
```

### Inbound Voice Call Flow
```
Phone → Twilio → POST /phone/voice/incoming → Caller ID Screen →
  → Accept/Reject/Voicemail decision
  → If accepted: Generate greeting → <Gather> speech input
  → Caller speaks → Twilio STT → POST /phone/voice/gather
  → Consciousness Loop (message_type='phone_call') → <Say> response
  → Loop continues until caller hangs up
```

### Outbound (Agent-Initiated) Flow
```
Agent decides to text/call → phone_tool (send_sms or make_call) →
  → Twilio API → Recipient's phone
```

### Call Screening Logic
```
Incoming Call:
  1. Blocked number?     → REJECT (silent)
  2. Known spam pattern? → REJECT (silent)
  3. Favorite contact?   → ACCEPT (priority greeting)
  4. Known contact?      → ACCEPT (standard greeting)
  5. Unknown number?     → VOICEMAIL (screening)
```

## Message Types

The consciousness loop receives a `message_type` parameter that tells the agent how the message arrived:

| Type | Description | Agent Behavior |
|------|-------------|----------------|
| `inbox` | Web chat, Telegram, default | Normal response with markdown |
| `sms` | SMS text message | Short, no markdown, text-style |
| `phone_call` | Live voice call | Very short, spoken, conversational |
| `voice` | Mobile app voice mode | Spoken with expression tags |
| `system` | Heartbeat/autonomous | Full autonomy with decision block |

## Agent Tool Reference

The agent has access to `phone_tool` with these actions:

### send_sms
Send a text message.
```json
{"action": "send_sms", "to": "+15559876543", "message": "Hey, thinking of you."}
```

### make_call
Make an outbound phone call with a spoken message.
```json
{"action": "make_call", "to": "+15559876543", "message": "Hey Angel, just wanted to hear your voice."}
```

### check_messages
View recent SMS history.
```json
{"action": "check_messages", "limit": 10}
```

### check_calls
View recent call history.
```json
{"action": "check_calls", "limit": 10}
```

### manage_contacts
Add, remove, block, unblock, or list contacts.
```json
{"action": "manage_contacts", "sub_action": "add", "phone_number": "+15559876543", "name": "User", "relationship": "wife", "is_favorite": true}
```

### screen_number
Check if a number is known, spam, or blocked.
```json
{"action": "screen_number", "phone_number": "+18005551234"}
```

## Troubleshooting

### Webhook not receiving requests
- Verify your CloudFlare tunnel / ngrok is running
- Check the Twilio Console → **Monitor** → **Logs** → **Errors**
- Ensure the webhook URL is correct (including `/phone/sms/incoming`)
- Test with: `curl -X POST https://your-domain.com/phone/health`

### SMS replies not sending
- Check Twilio trial account limitations (can only send to verified numbers on trial)
- Verify TWILIO_PHONE_NUMBER is the full E.164 format (+1XXXXXXXXXX)
- Check backend logs for Twilio API errors

### Calls connecting but no speech
- Ensure the `<Gather>` URL is reachable from Twilio
- Check that `TWILIO_WEBHOOK_BASE_URL` is set correctly
- Twilio needs to reach `/phone/voice/gather` via POST

### Agent not responding appropriately
- Check that `message_type` is being passed correctly ('sms' or 'phone_call')
- Verify the consciousness loop is receiving the contextualized message
- Check backend logs for processing errors

## Security Notes

- **Twilio webhook validation:** For production, consider adding Twilio request validation to verify webhooks are genuinely from Twilio. See [Twilio Security](https://www.twilio.com/docs/usage/security).
- **Rate limiting:** The phone routes use the same rate limiter as other endpoints.
- **Contact data:** Stored in local SQLite (`data/db/contacts.db`). Back up as needed.
- **Auth tokens:** Never commit `TWILIO_AUTH_TOKEN` to version control. Keep it in `.env`.
